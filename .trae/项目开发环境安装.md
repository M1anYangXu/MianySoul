### 1. **核心模型设计**

```prisma
model AlgorithmExperiment {
  id          String   @id @default(uuid())
  name        String
  description String?
  language    String   // javascript, python, typescript
  code        String   @db.Text
  inputData   String?  @db.Text  // 输入参数
  status      String   @default("draft") // draft, running, success, failed, timeout
  output      String?  @db.Text  // 执行结果
  error       String?  @db.Text  // 错误信息
  duration    Int?     // 执行耗时(ms)
  memoryUsage Int?     // 内存占用(KB)
  userId      String
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  user        User     @relation(fields: [userId], references: [id])
  runs        AlgorithmRun[]

  @@index([userId])
  @@index([language])
}

model AlgorithmRun {
  id          String   @id @default(uuid())
  experimentId String
  status      String   // running, success, failed, timeout
  output      String?  @db.Text
  error       String?  @db.Text
  duration    Int?
  memoryUsage Int?
  startedAt   DateTime @default(now())
  finishedAt  DateTime?

  experiment  AlgorithmExperiment @relation(fields: [experimentId], references: [id])

  @@index([experimentId])
}
```

### 2. **API 设计（RESTful）**

| 方法   | 路径                           | 功能                   |
| ------ | ------------------------------ | ---------------------- |
| GET    | `/api/algorithm`               | 列出我的所有实验       |
| POST   | `/api/algorithm`               | 创建新实验             |
| GET    | `/api/algorithm/:id`           | 获取实验详情           |
| PUT    | `/api/algorithm/:id`           | 更新实验代码/配置      |
| DELETE | `/api/algorithm/:id`           | 删除实验               |
| POST   | `/api/algorithm/:id/run`       | 执行实验（同步）       |
| POST   | `/api/algorithm/:id/run-async` | 异步执行（用于长任务） |
| GET    | `/api/algorithm/:id/runs`      | 查看历史运行记录       |
| GET    | `/api/algorithm/runs/:runId`   | 获取单次运行结果       |
| POST   | `/api/algorithm/format`        | 代码格式化（语法检查） |

### 3. **执行器设计（最关键）**

```typescript
// apps/backend/src/services/algorithm-runner.ts
interface RunOptions {
  code: string;
  language: "javascript" | "typescript" | "python";
  input?: string;
  timeout?: number; // 默认 5000ms
  memoryLimit?: number; // 默认 128MB
}

interface RunResult {
  status: "success" | "failed" | "timeout";
  output: string;
  error?: string;
  duration: number;
  memoryUsage: number;
}
```

**执行方案对比：**

| 方案                        | 优点           | 缺点               | 适用     |
| --------------------------- | -------------- | ------------------ | -------- |
| **子进程 + Docker**         | 最安全、隔离好 | 资源消耗大、启动慢 | 长期方案 |
| **子进程 + node:vm**        | 轻量、快速     | 隔离性差           | 当前推荐 |
| **Web Worker（前端）**      | 零后端压力     | 浏览器性能有限     | 简单算法 |
| **第三方 API（如 Judge0）** | 开箱即用       | 依赖外部、定制差   | 不想自建 |

**推荐：先用 `child_process` + Node.js `vm` 模块**，后期需要多语言时再升级到 Docker。

### 4. **安全与资源控制**

```typescript
// 资源限制
- 超时：默认 5 秒，最大 30 秒
- 内存：默认 128MB
- 输出大小：限制 1MB（防爆栈）
- 并发：单用户最多 3 个同时任务
- 黑名单：禁止 require('fs')、child_process 等敏感模块
- 沙箱：使用 VM2 或 isolated-vm 隔离执行环境
```

### 5. **支持的语言（分阶段）**

**第一阶段（现在）：**

- JavaScript / TypeScript（Node.js 子进程）
- 内置常用算法库：lodash、mathjs

**第二阶段：**

- Python（python-shell）
- C/C++（Judge0 API 或自建）

### 6. **特色功能**

- **📈 性能分析**：记录执行时间、内存占用、CPU 使用
- **🔄 版本管理**：每次保存自动生成快照，可回滚
- **📝 测试用例**：类似 LeetCode，可预设多个测试用例
- **🏆 挑战模式**：内置经典算法题（排序、查找、DP）
- **📤 分享**：生成分享链接，他人可 fork

### 7. **前端体验建议**

- Monaco Editor（VSCode 同款）

- 实时语法检查、自动补全
